Documentación del Editor pixelWallE

Bienvenido a la documentación del editor pixelWallE. Aquí encontrarás una guía completa para empezar a utilizar el editor, desde la carga de archivos hasta la exportación de tu arte pixel, incluyendo las funciones únicas como la conversión de imágenes a código.

pixelWallE está diseñado para ser intuitivo para principiantes, permitiéndote enfocarte en la creatividad mientras el editor maneja los detalles técnicos.

Cargar y Guardar Archivos

Gestionar tus proyectos de pixel art es fundamental. El editor pixelWallE ofrece opciones sencillas para abrir archivos existentes, crear nuevos y guardar tu progreso.

Abrir Archivos Existentes

Para continuar trabajando en un proyecto previamente guardado:

  • Ve al menú Archivo en la barra superior.
  • Selecciona la opción Abrir....
  • Navega hasta la ubicación de tu archivo .pxl (o el formato nativo de pixelWallE, si es diferente) y selecciónalo.

El contenido del archivo se cargará en el editor principal.

Crear Nuevos Archivos

Para empezar un nuevo proyecto desde cero:

  • Ve al menú Archivo en la barra superior.
  • Selecciona la opción Nuevo.

Esto abrirá un editor vacío, listo para que comiences a codificar o dibujar.

Guardar Archivos

Es crucial guardar tu trabajo regularmente para no perder el progreso.

  • Guardar: Si el archivo ya ha sido guardado antes, esta opción sobrescribirá el archivo existente con los cambios actuales. Puedes usar el atajo de teclado Ctrl + S (Windows/Linux) o Cmd + S (macOS).
  • Guardar como...: Esta opción te permite guardar el archivo actual con un nuevo nombre o en una ubicación diferente. Es útil para guardar una copia, crear una nueva versión o guardar un archivo nuevo por primera vez. Al seleccionar esta opción, se abrirá un diálogo donde puedes elegir el nombre y la carpeta.

Formatos de Archivo Soportados

pixelWallE utiliza su propio formato de archivo (por ejemplo, .pxl) para guardar el código de tu arte pixel. Al abrir o guardar, asegúrate de utilizar este formato.

Consejo: Guarda tu trabajo con frecuencia. Configura tu editor para hacer copias de seguridad automáticas si la opción está disponible.

Convertir Fotos a Código

Una característica potente de pixelWallE es la capacidad de convertir imágenes de mapa de bits (como fotos o cualquier archivo .PNG, .JPG, etc.) en código pixelWallE. Esto te permite importar arte existente y ver cómo se representa en nuestro lenguaje, o usarlo como punto de partida para tus propias creaciones.

Propósito de la Función

Esta herramienta analiza los píxeles de una imagen de entrada y genera automáticamente líneas de código pixelWallE que, al ejecutarse en el editor, reproducirían esa imagen (o una versión aproximada, dependiendo de la complejidad y las opciones de conversión).

Cómo Realizar la Conversión

  1. Ve al menú Herramientas en la barra superior.
  2. Selecciona la opción Foto a Código... (o un nombre similar).
  3. Se abrirá una ventana o panel donde podrás seleccionar la imagen que deseas convertir (admite formatos comunes como PNG, JPG, BMP).
  4. Dentro de esta ventana, podrías encontrar opciones como:
    • Tamaño de Salida: Define las dimensiones (ancho x alto) del pixel art resultante en términos de píxeles de salida. Un tamaño menor simplificará la imagen.
    • Paleta de Colores: Podrías tener la opción de limitar los colores a una paleta específica para dar a la imagen un aspecto retro o estilizado.
    • Umbral/Contraste: Ajustes para influir en cómo los colores se mapean o agrupan.
  5. Configura las opciones a tu gusto y haz clic en Convertir o Generar Código.

El código pixelWallE generado aparecerá en un nuevo archivo del editor o reemplazará el contenido actual, dependiendo de las opciones.

Formato del Código Generado

El código generado típicamente consistirá en una serie de comandos para dibujar píxeles individuales o formas simples utilizando los colores detectados en la imagen. Por ejemplo, podría generar algo parecido a:

set_pixel(x=0, y=0, color="#FF0000");
set_pixel(x=1, y=0, color="#FFA500");
set_pixel(x=0, y=1, color="#FFA500");
set_pixel(x=1, y=1, color="#FFFF00");
# ... muchas más líneas para cada pixel o área ...
draw_rectangle(x=10, y=10, width=5, height=5, color="#0000FF"); # Podría agrupar píxeles

La complejidad y el formato exacto dependerán de la implementación, pero la idea es reproducir visualmente la imagen de origen.

Limitaciones y Consideraciones

  • La calidad del código y el parecido con la imagen original dependen en gran medida de la imagen de entrada y las opciones de conversión elegidas. Imágenes de baja resolución o con pocos colores funcionan mejor.
  • Convertir imágenes muy grandes o complejas puede generar una cantidad de código extremadamente larga.
  • La herramienta intenta interpretar la imagen; el resultado final puede requerir ajustes manuales en el código para perfeccionarlo.

Consejo: Experimenta con diferentes tamaños de salida y paletas de colores para obtener efectos interesantes. Utiliza imágenes con contornos definidos para mejores resultados.

Guardar Pixel Art como Imagen

Una vez que has creado tu obra maestra pixel a pixel, querrás exportarla como un archivo de imagen estándar para compartirla, usarla o publicarla.

La función de exportar te permite convertir el pixel art que ves en la previsualización en un archivo de imagen de mapa de bits.

Cómo Exportar

  1. Asegúrate de que la previsualización de tu pixel art es visible y está actualizada con los cambios que deseas exportar.
  2. Ve al menú Archivo (o Exportar si hay un menú separado para ello).
  3. Selecciona la opción Exportar como Imagen....
  4. Se abrirá un diálogo donde puedes configurar las opciones de exportación:
    • Formato de Imagen: Elige entre formatos comunes como PNG, JPG, BMP. PNG es ideal para pixel art ya que soporta transparencia y compresión sin pérdida.
    • Escala/Zoom: Puedes exportar la imagen a su tamaño real (1:1) o escalar cada pixel a un tamaño mayor (ej. 2x, 3x, 10x) para que el arte sea más visible sin perder la nitidez de los píxeles.
    • Ruta y Nombre del Archivo: Elige dónde guardar el archivo y qué nombre tendrá.
  5. Haz clic en Exportar o Guardar.

Tu pixel art se guardará como un archivo de imagen en la ubicación especificada.

Consejo: Para la mayoría de los casos de pixel art, se recomienda exportar a formato PNG con una escala mayor a 1x (ej. 4x o 8x) para que los píxeles individuales sean claramente visibles en pantallas de alta resolución. Evita JPG si tu arte tiene líneas nítidas o grandes áreas de color plano, ya que la compresión con pérdida puede introducir artefactos.

Uso de la Consola Integrada

La consola integrada en pixelWallE es una herramienta esencial para interactuar directamente con el editor, ejecutar scripts, ver mensajes de error detallados, la salida de tus programas y realizar tareas administrativas.

Cómo Abrir la Consola

La consola suele estar visible por defecto en un panel en la parte inferior del editor. Si no la ves, puedes abrirla y cerrarla desde el menú Ver o Ventanas, seleccionando Consola o Terminal.

Ejecutar Programas y Ver Salida

Puedes ejecutar tu código pixelWallE directamente desde la consola o usando el botón de "Ejecutar" en la interfaz (que internamente usa la consola). Los mensajes que tu código imprima (si el lenguaje pixelWallE lo permite) y cualquier error de ejecución aparecerán aquí.

También puedes escribir comandos directamente en la línea de entrada de la consola (si está disponible).

Ejemplos de Comandos Genéricos

Aquí hay algunos ejemplos de comandos que podrías usar en la consola (la disponibilidad y sintaxis exacta pueden variar en futuras versiones o configuraciones):

  • run mi_script.pxl: Ejecuta el archivo llamado mi_script.pxl en el editor actual.
  • clear: Limpia la salida de texto mostrada en la consola, útil para empezar de nuevo.
  • ls (o dir en algunos sistemas): Lista los archivos en el directorio de trabajo actual (útil para ver tus scripts guardados).
  • run_with_args mi_script.pxl arg1 arg2: Ejecuta mi_script.pxl pasándole argumentos (arg1, arg2) que tu script podría usar para modificar su comportamiento.

Nota: Estos son ejemplos de comandos. La lista completa de comandos disponibles y su sintaxis exacta se encuentra en la Documentación de la Consola.

Consejo: Mantén la consola abierta mientras trabajas. Es la primera línea de defensa para depurar problemas y entender qué está haciendo tu código paso a paso.

Botones Laterales

En el borde del editor (comúnmente en la parte izquierda o derecha), encontrarás una barra de botones que te dan acceso rápido a funcionalidades clave sin necesidad de usar los menús o atajos de teclado. La ubicación y el diseño exacto pueden variar ligeramente.

Aquí están algunos de los botones más importantes que encontrarás:

  • Cambiar Tamaño de la Interfaz: Representado usualmente por un icono de flechas apuntando hacia afuera () o hacia adentro (). Permite ajustar el tamaño de los paneles del editor (como el código, la previsualización, la consola) o alternar entre modo de pantalla completa/compacta.

  • Deshacer (Undo): Un icono de flecha curvada hacia la izquierda (). Deshace la última acción realizada en el código o en la previsualización (si aplica).

  • Rehacer (Redo): Un icono de flecha curvada hacia la derecha (). Rehace la última acción que fue deshecha.

  • Redimensionar Elemento: Podría ser un icono de un cuadrado con flechas en las esquinas () o un icono similar. Este botón es específico si el editor permite seleccionar y redimensionar elementos gráficos directamente en la previsualización en lugar de solo a través del código. (Aclara si aplica a elementos gráficos en previsualización).

    (Nota: Esta función puede no estar presente o ser diferente dependiendo de la versión del editor).

  • Ejecutar Código (Run): Un icono de un triángulo de "play" (). Al hacer clic, ejecuta el código actual en el editor y muestra el resultado en la previsualización y la consola.

Consejo: Estos botones son accesos directos convenientes para las operaciones más frecuentes. Ubícalos y úsalos para no tener que navegar por menús constantemente.

Errores en Tiempo Real

pixelWallE te ayuda a detectar y corregir errores en tu código a medida que escribes, mucho antes de que intentes ejecutarlo. Esto se llama "detección de errores en tiempo real".

Cómo se Muestran los Errores

Los errores y advertencias se muestran de varias maneras:

  • Subrayado en el Código: Las partes del código que contienen un error (como una palabra mal escrita, un paréntesis que falta, etc.) se subrayan con una línea de color (típicamente rojo para errores, amarillo para advertencias).
  • Mensajes Emergentes (Tooltips): Al pasar el ratón sobre el código subrayado, aparecerá una pequeña caja de texto (tooltip) con una descripción más detallada del problema.
  • Panel de Problemas/Errores: El editor tiene un panel dedicado (a menudo en la parte inferior, junto a la consola) que lista todos los errores y advertencias encontrados en el archivo actual. Al hacer clic en un elemento de la lista en este panel, el editor te llevará directamente a la línea de código donde se encuentra el problema.

Tipos de Errores

El editor puede detectar:

  • Errores de Sintaxis: Son problemas relacionados con la gramática del lenguaje. Por ejemplo, olvidar una coma, un paréntesis, escribir mal un comando. El código no se puede entender.
  • Errores Semánticos Básicos: Problemas relacionados con el significado del código. Por ejemplo, intentar usar un color que no existe en la paleta permitida, o usar un comando con un tipo de valor incorrecto (ej. pasar texto donde se espera un número).

Interpretar Mensajes de Error

Los mensajes de error suelen indicar:

  • Tipo de Problema: Indica si es un "Error" (que impedirá que el código se ejecute) o una "Advertencia" (un posible problema que el código podría ejecutar pero no es ideal).
  • Descripción: Una breve explicación de qué está mal (ej. "se esperaba un ';' al final de la línea", "color desconocido '#INVALID'").
  • Ubicación: La línea y columna exacta donde el problema fue detectado.

Consejos para Solucionar Errores:

  • Lee el mensaje de error con atención. A menudo te dice exactamente qué falta o qué está mal.
  • Empieza por el primer error listado. Corregir un error al principio del código a veces soluciona varios errores posteriores.
  • Revisa la línea indicada y las líneas justo antes y después. El error reportado puede ser consecuencia de algo en una línea anterior.
  • Consulta la Documentación del Lenguaje pixelWallE para verificar la sintaxis correcta de los comandos y la especificación de colores.

Consejo: No te frustres con los errores. Son una parte normal del proceso de codificación y el editor te está ayudando a encontrarlos rápidamente.
