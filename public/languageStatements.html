Comandos (Sentencias) y Funciones

Esta sección detalla las acciones que puedes programar en pixelWallE. Los comandos son las instrucciones principales para dibujar y manipular el lienzo, mientras que las funciones te permiten obtener información sobre el estado del editor o el contenido del lienzo.

Comandos (Sentencias)

Los comandos son sentencias ejecutables que realizan una acción, como dibujar una forma, cambiar el color de la brocha o mover un punto de referencia. No devuelven un valor directamente y se usan como líneas de código independientes.

SetPixel(x, y, color)

Dibuja un único píxel en las coordenadas especificadas (x, y) usando el color dado. Las coordenadas son relativas a la esquina superior izquierda del lienzo (0,0).

  • x (Número): Coordenada X del píxel.
  • y (Número): Coordenada Y del píxel.
  • color (Cadena): Color del píxel en formato de nombre ("red") o Hexadecimal ("#FF0000").
SetPixel(10, 15, "blue");
SetPixel(0, 0, "#FF0000"); # Dibuja un pixel rojo en la esquina superior izquierda

DrawCircle(x, y, radius, color)

Dibuja un círculo sólido con el centro en (x, y), el radius y el color especificados.

  • x (Número): Coordenada X del centro del círculo.
  • y (Número): Coordenada Y del centro del círculo.
  • radius (Número): El radio del círculo en píxeles.
  • color (Cadena): Color del círculo. (Puede ser opcional si el lenguaje usa un color de brocha actual).
DrawCircle(50, 50, 20, "red"); # Dibuja un círculo rojo en el centro del lienzo (asumiendo un lienzo 100x100)
DrawCircle(10, 10, 5); # Dibuja un círculo de radio 5 con el color de brocha actual

Ejemplo Visual:

Imagen aquí: Ejemplo de DrawCircle

DrawLine(x1, y1, x2, y2, color)

Dibuja una línea recta desde el punto inicial (x1, y1) hasta el punto final (x2, y2) con el color dado.

  • x1 (Número): Coordenada X del punto inicial.
  • y1 (Número): Coordenada Y del punto inicial.
  • x2 (Número): Coordenada X del punto final.
  • y2 (Número): Coordenada Y del punto final.
  • color (Cadena): Color de la línea. (Puede ser opcional).
DrawLine(0, 0, 99, 99, "black"); # Dibuja una diagonal en un lienzo 100x100
DrawLine(GetActualX(), GetActualY(), GetActualX() + 10, GetActualY(), "white"); # Dibuja una línea horizontal desde la brocha actual

DrawRectangle(x, y, width, height, color)

Dibuja un rectángulo sólido con su esquina superior izquierda en (x, y), con el width, height y color especificados.

  • x (Número): Coordenada X de la esquina superior izquierda.
  • y (Número): Coordenada Y de la esquina superior izquierda.
  • width (Número): Ancho del rectángulo en píxeles.
  • height (Número): Alto del rectángulo en píxeles.
  • color (Cadena): Color del rectángulo. (Puede ser opcional).
DrawRectangle(10, 10, 30, 20, "#00FF00"); # Dibuja un rectángulo verde
DrawRectangle(0, 0, GetCanvasSize() / 2, GetCanvasSize() / 2, "blue"); # Dibuja un cuadrado azul en la esquina superior izquierda

Fill(color)

Rellena todo el área visible del lienzo con el color especificado.

  • color (Cadena): El color para rellenar el lienzo.
Fill("white"); # Borra el lienzo pintándolo de blanco
Fill("#282c34"); # Rellena con el color de fondo oscuro

GoTo(x, y)

Mueve el punto de referencia o la "brocha" actual del editor a las coordenadas (x, y) sin dibujar. Comandos posteriores que dependen de la posición actual (como DrawLine si se usa desde la brocha) comenzarán desde este nuevo punto.

  • x (Número): Coordenada X de destino.
  • y (Número): Coordenada Y de destino.
GoTo(50, 50); # Mueve la brocha al centro
DrawLine(GetActualX(), GetActualY(), GetActualX() + 20, GetActualY(), "red"); # Dibuja una línea roja horizontal desde (50, 50)

Nota: La gramática original sugería GoTo(true, "label"), lo que implicaría control de flujo (saltos condicionales a etiquetas). Si esa es la implementación real, esta descripción sería incorrecta. Basándonos en GetActualX/Y, interpretamos GoTo como mover la brocha. Si hay soporte para etiquetas y goto, esa funcionalidad se describiría aquí también o en una sección aparte de Control de Flujo.

Size(size)

Establece el tamaño de la "brocha" actual. Esto puede afectar el grosor de las líneas o el tamaño de los puntos dibujados por otros comandos si el lenguaje lo soporta.

  • size (Número): El nuevo tamaño o grosor. Un valor de 1 suele ser un solo píxel.
Size(3); # Establece un grosor de 3 píxeles
Color("black");
# Si DrawLine respeta el tamaño:
DrawLine(10, 20, 40, 50); # Dibuja una línea negra de 3 píxeles de grosor

Nota: La implementación exacta de cómo Size afecta a otros comandos depende del editor.

Spawn(x, y)

Similar a GoTo, este comando podría usarse para establecer un punto de inicio, quizás para un agente o un sistema de partículas, en las coordenadas (x, y).

  • x (Número): Coordenada X de spawn.
  • y (Número): Coordenada Y de spawn.
Spawn(GetCanvasSize() / 2, GetCanvasSize() / 2); # Inicia algo en el centro

Nota: El propósito exacto de este comando depende de otras funcionalidades del editor que utilicen un concepto de "spawn". Podría ser idéntico a GoTo en su efecto sobre la "brocha" principal, o afectar a otros elementos.

Color(color)

Establece el color de la "brocha" o el color actual para los comandos de dibujo subsiguientes que no especifican un color explícitamente (ej. DrawCircle(x, y, r) en lugar de DrawCircle(x, y, r, color)).

  • color (Cadena): El nuevo color de brocha.
Color("green"); # Establece la brocha a verde
DrawCircle(20, 20, 10); # Dibuja un círculo verde
Color("#FF00FF"); # Cambia la brocha a magenta
DrawRectangle(40, 40, 20, 20); # Dibuja un rectángulo magenta

Funciones

Las funciones son bloques de código que realizan una tarea o cálculo y devuelven un valor. Este valor puede ser utilizado en expresiones, asignado a variables o pasado como argumento a comandos o a otras funciones.

Se llaman por su nombre seguido de paréntesis (). Si requieren información para operar, esta se pasa como argumentos dentro de los paréntesis.

GetActualX()

Devuelve la coordenada X actual del punto de referencia o la "brocha" del editor.

  • No toma argumentos.
  • Devuelve: Número.
currentX = GetActualX();
SetPixel(GetActualX() + 1, GetActualY(), "red"); # Dibuja un pixel a la derecha de la brocha actual

GetActualY()

Devuelve la coordenada Y actual del punto de referencia o la "brocha" del editor.

  • No toma argumentos.
  • Devuelve: Número.
currentY = GetActualY();
GoTo(10, GetActualY()); # Mueve la brocha a la columna 10, manteniendo la misma fila

GetCanvasSize()

Devuelve el tamaño del lienzo. Si el lienzo es cuadrado, devuelve un único número para el ancho y el alto. Si puede ser rectangular, podría devolver el ancho o requerir argumentos o funciones separadas (como GetCanvasWidth() y GetCanvasHeight()).

  • No toma argumentos.
  • Devuelve: Número (asumiendo lienzo cuadrado o devolviendo el ancho).
size = GetCanvasSize();
Fill("#CCCCCC"); # Rellena el lienzo completo
DrawRectangle(0, 0, GetCanvasSize(), GetCanvasSize(), "black"); # Dibuja un borde alrededor del lienzo

GetColorCount(color, x, y, width, height)

Cuenta cuántos píxeles dentro de un área rectangular especificada tienen el color exacto dado.

  • color (Cadena): El color a buscar.
  • x (Número): Coordenada X de la esquina superior izquierda del área de búsqueda.
  • y (Número): Coordenada Y de la esquina superior izquierda del área de búsqueda.
  • width (Número): Ancho del área de búsqueda.
  • height (Número): Alto del área de búsqueda.
  • Devuelve: Número entero (la cantidad de píxeles encontrados).
numRedPixels = GetColorCount("red", 0, 0, 100, 100); # Cuenta píxeles rojos en todo un lienzo 100x100
numBlueArea = GetColorCount("blue", 10, 10, 30, 30); # Cuenta píxeles azules en un área específica

Nota: Los argumentos de área pueden ser opcionales o tener valores por defecto para contar en todo el lienzo.

IsBrushColor(color)

Verifica si el color de brocha actual es igual al color especificado.

  • color (Cadena): El color con el que comparar.
  • Devuelve: Booleano (true si coinciden, false si no).
if (IsBrushColor("blue")) {
    # Código que solo se ejecuta si la brocha es azul
    DrawCircle(GetActualX(), GetActualY(), 5);
}

(Asume la existencia de sentencias de control de flujo como if).

IsBrushSize(size)

Verifica si el tamaño de brocha actual es igual al size especificado.

  • size (Número): El tamaño con el que comparar.
  • Devuelve: Booleano.
isSize5 = IsBrushSize(5); # isSize5 será true o false

IsCanvasColor(color)

Verifica si todos los píxeles en el lienzo tienen el color especificado.

  • color (Cadena): El color a verificar.
  • Devuelve: Booleano.
if (IsCanvasColor("white")) {
    Fill("black"); # Si el lienzo está todo blanco, pintarlo de negro
}

(Este comando puede ser menos eficiente en lienzos muy grandes).

IsColor(color, x, y)

Verifica si el píxel individual en las coordenadas (x, y) tiene el color especificado.

  • color (Cadena): El color a verificar.
  • x (Número): Coordenada X del píxel.
  • y (Número): Coordenada Y del píxel.
  • Devuelve: Booleano.
if (IsColor("#00FF00", 10, 10)) {
    SetPixel(11, 11, "black"); # Si el pixel en (10,10) es verde, dibuja uno negro en (11,11)
}

GetPixelColor(x, y)

Devuelve el color exacto del píxel en las coordenadas (x, y).

  • x (Número): Coordenada X del píxel.
  • y (Número): Coordenada Y del píxel.
  • Devuelve: Cadena (el color del píxel, probablemente en formato Hexadecimal).
pixelColor = GetPixelColor(5, 5); # Obtiene el color del pixel en (5,5)
Color(pixelColor); # Establece ese color como brocha actual
DrawCircle(6, 6, 1); # Dibuja un pequeño círculo con el color del pixel (5,5)

Variables

Puedes declarar variables para almacenar valores (números, cadenas, booleanos, listas) y reutilizarlos a lo largo de tu script. Se declaran usando un identificador (nombre de la variable) y se les asigna un valor inicial usando el operador =.

myNumber = 10;
message = "Drawing complete!";
isDone = true;
myColors = ["red", "green"]; # Declarando con un literal de lista

# Puedes reasignar variables
myNumber = myNumber + 5; # myNumber ahora vale 15
message = "Updated " + message;

El tipo de una variable se determina por el valor que se le asigna inicialmente. Las variables de lista requieren la sintaxis List<Tipo> Nombre = ....

Las variables pueden ser usadas en cualquier lugar donde se espere una expresión del tipo correspondiente.

Control de Flujo

El control de flujo te permite alterar la secuencia normal de ejecución de tu código, permitiendo crear bucles o ejecutar código condicionalmente.

Etiquetas (Labels)

Las etiquetas marcan puntos específicos en tu código a los que puedes saltar usando la sentencia goto. Una etiqueta consiste en un identificador seguido de dos puntos :.

# Inicio del script
Setup: # Esto es una etiqueta
    Fill("black");
    GoTo(10, 10);
    DrawCircle(GetActualX(), GetActualY(), 5, "white");

DrawLoop: # Otra etiqueta, quizás para un bucle
    # Código de dibujo repetitivo aquí
    # ...

    # Si se cumple una condición, salta a otra etiqueta
    # if (condition) goto EndScript; # Ejemplo conceptual de goto condicional
    goto DrawLoop; # Bucle infinito simple

EndScript: # Etiqueta para finalizar
    # Código de limpieza o finalización
    Fill("green");

Nota: El uso de goto y etiquetas puede hacer el código difícil de leer y mantener. Considere usar estructuras de bucle o condicionales si el lenguaje las soporta (como if, loop, while).

goto Etiqueta

Transfiere incondicionalmente la ejecución del programa a la sentencia inmediatamente posterior a la etiqueta especificada.

  • Etiqueta (Identificador): El nombre de la etiqueta a la que saltar.
    # Algunas sentencias
    goto MyLoop;

    # Este código se omite
    SetPixel(0,0,"red");

MyLoop:
    # El programa continúa desde aquí
    # ...

Si la gramática GoTo(true, "label") se refiere a un salto condicional, entonces la sintaxis sería diferente y este comando goto simple podría no existir, o GoTo podría sobrecargarse para ambos propósitos (mover brocha Y saltar). Aclarar la implementación real es importante.

if (condición) { ... } [else { ... }]

Ejecuta un bloque de código si una condición booleana es verdadera. Opcionalmente, puedes proporcionar un bloque else que se ejecuta si la condición es falsa.

  • condición (Expresión Booleana): La expresión que se evalúa.
if (GetActualX() > 50 && GetActualY() > 50) {
    # Si la brocha está en el cuadrante inferior derecho
    Color("yellow");
} else {
    # Si está en cualquier otro lugar
    Color("cyan");
}

if (IsColor("black", 10, 10)) {
    # Si el pixel es negro, salta a una etiqueta (usando goto como ejemplo)
    goto FoundBlack;
}

Nota: La sintaxis exacta para bloques de código ({} o indentación) y la estructura else pueden variar.
